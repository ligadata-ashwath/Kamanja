  10. def getModelInstanceFactory(modelDef: ModelDef, nodeContext: NodeContext, loaderInfo: KamanjaLoaderInfo, jarPaths: Set[String]): ModelInstanceFactory

    Instantiate a model instance factory for the supplied modelDef.

    Instantiate a model instance factory for the supplied modelDef. The model instance factory can instantiate instances of the model described by the ModelDef.

    modelDef

    the metadatata object for which a model factory instance will be created.

    nodeContext

    the NodeContext that provides access to model state and kamanja engine services.

    loaderInfo

    the engine's custom loader to use if/when instances of the model and the dependent jars are to be loaded

    jarPaths

    a Set of Paths that contain the dependency jars required by this factory instance to be created

    returns

    a ModelInstanceFactory or null if bogus information has been supplied. Fixme: Passing the jarPaths as a Set suggests that there is no search order in the path. Should we have an ordered list instead to allow for alternate implementations where the lib/application directory is first followed by the lib/system directory? This could be fruitfully used to drop in a fixed implementation ahead of a broken one... this all assumes there is a command to force reload of jars for a given model.
